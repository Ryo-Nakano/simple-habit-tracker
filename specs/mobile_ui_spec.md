# モバイル対応UI

## 概要
- スマートフォンでも快適に操作できるよう、UIをレスポンシブ対応し、タッチ操作を最適化する
- PC向けのクリック中心のUIから、モバイルのタッチ中心のUIへ改善する

## 仕様
### 機能要件
- モバイル端末（幅狭画面）での表示崩れを防ぐ
- タッチターゲット（ボタン、チェックボックス等）のサイズを拡大し、操作しやすくする

### UI/メッセージ
- 特になし（既存の機能を維持）

### 制約・前提
- Tailwind CSSを使用
- Reactコンポーネントを修正

## 実装計画
### 変更するファイル
- `src/client/components/TaskItem.tsx`: チェックボックスと編集ボタンのタッチエリア拡大
- `src/client/components/AddTaskForm.tsx`: ボタンのサイズ調整、モバイル時のレイアウト調整
- `src/client/components/Dashboard.tsx`: 全体の余白調整（必要であれば）
- `src/client/components/Heatmap.tsx`: 横スクロールの明示（必要であれば）

### 処理フロー
1. 各コンポーネントのCSSクラス（Tailwind）を調整
2. `p-` や `w-`, `h-` を調整してタッチターゲットを44px以上確保することを目指す
3. レスポンシブクラス（`md:`, `lg:`）を活用し、PC表示への影響は最小限にする

### 技術的な判断・注意点
- 既存のPCレイアウトを大きく崩さないように、モバイルファーストではなく、現状のデスクトップUIをベースにモバイル調整を行うアプローチをとる（または必要に応じてモバイルファーストに書き換えるが、工数を抑えるため調整メインとする）
